=== IT-Sicherheit und Datenschutz

==== Zugriffsschutz und Berechtigungsmanagement

Der Zugriff auf {application} ist durch ein mehrstufiges Berechtigungssystem sowie technische Schutzmaßnahmen abgesichert:

- Anmeldung erfolgt ausschließlich über das vom Kunden gewählte und konfigurierte Anmeldeverfahren (über die lokale Benutzer-/Passwortverwaltung
(LDAP des Kunden, DB des Kunden), über Azure AD/Entra (OIDC/SAML), über API-Token für Integrationen)
- Passwortrichtlinien entsprechen den unternehmenseigenen IT-Sicherheitsvorgaben (Mindestlänge, Komplexität, regelmäßiger Wechsel).
- Der Zugriff auf sensible Daten und Administrationsfunktionen ist ausschließlich ausgewählten Benutzerrollen vorbehalten.
- Verschlüsselung des Datenverkehrs zwischen Client und Server durch TLS 1.2 oder höher.

==== Security und Compliance

===== TLS/HTTPS

Obligatorisch für Produktion. Zertifikate:

- Let's Encrypt (via Caddy)
- DNS-01 (API-Token Provider)
- Eigene Zertifikate (interne CA)

===== Firewall

**Ausgehende Verbindungen:**

- **Immer:** Docker Hub / CGS Registry
- **Je nach LLM-Provider:**
  - Azure OpenAI: `*.openai.azure.com`
  - AWS Bedrock: `bedrock-runtime.<region>.amazonaws.com`
  - Lokal: Interne Verbindung

===== Datenschutz

**Anwendungsdaten:** Ausschließlich auf Kundenserver

**LLM-Provider (Cloud):**

- Azure OpenAI / AWS Bedrock: Anfragen werden übertragen, aber nicht dauerhaft gespeichert
- **Empfehlung:** EU-Region wählen (DSGVO)

**Lokale Modelle:** Höchste DSGVO-Konformität

==== Protokollierung und Logging

Es werden umfangreiche Protokolle (Logs) zur Sicherstellung von Nachvollziehbarkeit und zum Erkennen sicherheitsrelevanter Ereignisse geführt:

- Anmeldung, Logout, fehlgeschlagene Anmeldeversuche und Passwortänderungen werden protokolliert.
- Änderungen an sensiblen Daten (z. B. Löschvorgänge, Rechtevergaben) werden detailliert, inkl. Zeitstempel und Benutzer, erfasst.
- Die Logfiles werden automatisch zentral archiviert und vor unbefugtem Zugriff geschützt.
- Logdaten werden mindestens 12 Monate aufbewahrt und regelmäßig auf Sicherheitsvorfälle ausgewertet.
- Bei Verdacht auf einen Datenschutz- oder Sicherheitsvorfall erfolgt eine sofortige Meldung an den IT-Sicherheitsbeauftragten.

Die Protokollierungs- und Löschkonzepte werden jährlich überprüft und bei Bedarf gemäß rechtlichen Vorgaben angepasst.
