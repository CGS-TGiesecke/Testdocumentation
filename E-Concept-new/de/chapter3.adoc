

=== Technisches Grobkonzept
==== Architektur

Webanwendung mit:

Frontend (Browser) +
Backend (API) +
Datenbanken der ERP Systeme

==== Technologien

Das System besteht aus 3 Bausteinen:

*A) {application} Benutzeroberfläche*

* Web-App, in die der Benutzer Fragen eingibt
* Auswahl des Agenten
* Empfang der Ergebnisse und Darstellung in Tabellen und CSV Download
* später: Übergabe der Ergebnisse an PowerBI

*B) SQL-Agent / MCP-Server*

Der intelligente Kern:

* versteht natürliche Sprache
* analysiert automatisch: 
** Tabellen
** Beziehungen
** Spalten
* erzeugt korrekte, optimierte SQL-Abfragen
* interpretiert Ergebnisse und erklärt sie
* Später: Übergabe der Ergebnisse an MS Fabric

*C) Sicherer Relay Server beim Kunden*

Wichtiger Punkt für Datenschutz & Sicherheit:

* diese Komponente wird beim Kunden gehostet
* der Relay-Server führt die SQL-Abfragen direkt auf der Kundendatenbank aus
* nur die Ergebnisse werden zurückgeschickt
* Authentifizierung über API Keys
* keine direkten DB-Verbindungen nach außen

*Damit bleibt die gesamte Datenhoheit beim Kunden.*

==== Schnittstellen

Je ein Relay-Server zur Verbindung mit der DB je ERP-System +
{application} zu allen Relay-Servern
Datenübergabe an MS Factory +
Datenübergabe an PowerBI

=== Risiken & Annahmen
*Risiken*

Unklare interne Prozesse beim Kunden +
Qualität der Rohdaten in den ERP-System beim Kunden +
Änderungswünsche während der Entwicklung

*Annahmen*

{application} stellt *kein LLM* bereit. Der Kunde muss mindestens einen LLM-Provider selbst einrichten und betreiben. +
Kunde stellt Inhalte zur Verfügung (Prozesse, Anwendungsfälle) +
Alle Nutzer verfügen über aktuelle Browser

=== Sicherheits- und Compliance-Konzept

* Read-Only Zugriff - keine Änderungen an Daten
* SQL Injection Schutz
* Validierte Tabellen- und Spaltennamen
* Authentifizierung via API-Key
* Audit-Trail über ausgeführte Queries (Logging)
* Daten verlassen den Kunden nicht ungefiltert
* Keine Übertragung personenbezogener Daten ohne Zustimmung (PII-Handling)
