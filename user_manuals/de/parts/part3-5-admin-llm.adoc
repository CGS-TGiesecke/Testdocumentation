
==== LLMs und Einbettungsmodelle 

Die Seite listet alle LLM‑Provider die u.a. im Chat verwendet werden können.

image::../images/Abbildung-22.jpg[Administration - LLMs und Einbettungsmodelle, title="Administration - LLMs und Einbettungsmodelle", width=600]

Neue Provider und Modelle können angelegt oder, nach einer Sicherheitsabfrage, wieder entfernt werden. Dabei kann vorausgewählt werden ob AWS, Azure oder lokaler Provider erstellt werden soll.
 
image::../images/Abbildung-32.jpg[Administration - LLMs und Einbettungsmodelle - Provider anlegen, title="Administration - LLMs und Einbettungsmodelle - Provider anlegen", width=250]

Über den Pfeil am linken Rand kann die Konfiguration geöffnet werden und die notwendigen Modelle können erstellt oder,
nach einer bestätigten Sicherheitsabfrage, auch entfernt werden.

image::../images/Abbildung-24-azure.jpg[Administration - LLMs und Einbettungsmodelle - Azure Model bearbeiten, title="Administration - LLMs und Einbettungsmodelle - Azure Model bearbeiten", width=600]

Für die Anlage von Azure Provider Modellen werden folgende Eingaben benötigt:

* Key Name – Identifizierender Name eines API-Schlüssels z.B. "azure-key"
* Display Name – nutzerfreundlich dargestellter Anzeigename z.B. "Azure Modelle" (wird in der Übersicht in {application} angezeigt)
* API Key - Geheimnis zur Authentifizierung von API-Nutzern z.B. "5qjEqqufNM5XxXxXxXxXxXxXXXXXXXXXXXX"
* API Endpoint – Konkrete URL zu einer Funktion oder Ressource in einer API z.B. "https://assist-azure.openai.azure.com/"
* API Version – Versionsnummer/-bezeichnung einer API z.B. "2025-04-01-preview"

image::../images/Abbildung-24-aws.jpg[Administration - LLMs und Einbettungsmodelle - AWS Model bearbeiten, title="Administration - LLMs und Einbettungsmodelle - AWS Model bearbeiten", width=600]

Für die Anlage von AWS Provider Modellen werden folgende Eingaben benötigt:

* Key Name – Identifizierender Name eines API-Schlüssels z.B. "aws-key"
* Display Name – nutzerfreundlich dargestellter Anzeigename z.B. "AWS Modelle" (wird in der Übersicht in {application} angezeigt)
* IAM Access Key - – Benutzername für API-Zugriffe, eindeutig zugewiesener Identifikator z. B. "AKIAIOSFODNN7EXAMPLE"
* IAM Secret Access Key - – Geheimteil eines Schlüsselpaars, funktioniert wie ein Passwort z.B. "wJalrXUtnFEMI/K7MDENG..." (darf nicht öffentlich werden)
* Region - Geografisches Gebiet/Rechenzentrum, in dem Cloud-Ressourcen laufen z. B. "eu-central-1"

image::../images/Abbildung-24-lokal.jpg[Administration - LLMs und Einbettungsmodelle - Lokales Model bearbeiten, title="Administration - LLMs und Einbettungsmodelle - Lokal Model bearbeiten", width=600]

Für die Anlage von Lokalen Modellen werden folgende Eingaben benötigt:

* Key Name – Identifizierender Name eines API-Schlüssels z.B. "entwicklungs-key"
* Display Name – nutzerfreundlich dargestellter Anzeigename z.B. "Lokale Modelle" (wird in der Übersicht in {application} angezeigt)
* API Key - Geheimnis zur Authentifizierung von API-Nutzern z.B. "dev-api-key-1234567890abcdef"
* API Endpoint – Konkrete URL zu einer Funktion oder Ressource in einer API z.B. "http://localhost:8080/api/v1/users"
* API Version – Versionsnummer/-bezeichnung einer API z.B. "2026-01-01-latest"
