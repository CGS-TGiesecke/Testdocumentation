=== Assist Administration

Within Assist Administration, there is the possibility to centrally manage all system-relevant data. Here, users and roles, 
the document storage folders (RAG), interface configurations, system prompts, licenses, and the system-side 
stored use cases can be managed.
Access to these administration functions is granted via a system-defined, pre-configured role.

==== User Management

On this site, users can be created and edited. 

image::../images/Abbildung-16.jpg[Administration - User Management, title="Administration - User Management", width=400]

===== Edit User

The use of user management depends on the configured authentication method.

===== OpenID Connect Authentication with Microsoft Entra ID
Adding and editing users via the {application} interface is *not* possible.
Password management via Microsoft.
Role assignment for {application} via the {application} interface is possible.
Users cannot be deactivated and can only be removed from the overview table after confirming a security prompt.

===== SAML Authentication with Microsoft Entra ID
Adding and editing users via the {application} interface is *not* possible.
Password management via Microsoft.
Role assignment for {application} via the {application} interface is possible.
Users cannot be deactivated and can only be removed from the overview table after confirming a security prompt.

===== Local User/Password Management
Adding and editing users via the {application} interface is possible.
Password management via the {application} interface is possible.
Role assignment for {application} via the {application} interface is possible.
Users cannot be deactivated and can only be removed from the overview table after confirming a security prompt.
