
==== LLMs and Embedding Models

All LLM providers stored in the system are listed and managed here. These providers supply access to Large Language Models (LLMs) and maintain the infrastructure required to operate advanced AI language models.

image::../images/Abbildung-22.jpg[Administration - LLMs and Embedding Models, title="Administration - LLMs and Embedding Models", width=500]

New providers and models can be created or, after a security prompt, removed again. You can also preselect whether an AWS, Azure, or local provider should be created.

image::../images/Abbildung-32.jpg[Administration - LLMs and Embedding Models - Create Provider, title="Administration - LLMs and Embedding Models - Create Provider", width=250]

By clicking the arrow on the left, the provider configuration can be opened, and required models may be created or removed (removal also requires a confirmed security prompt).

image::../images/Abbildung-24-aws.jpg[Administration - LLMs and Embedding Models - Edit AWS Model, title="Administration - LLMs and Embedding Models - Edit AWS Model", width=500]

For creating Azure Provider Models, the following inputs are required:

* Key Name – Identifying name of an API key, e.g., "azure-key"
* Display Name – User-friendly display name, e.g., "Azure Models" (displayed in the overview in {application})
* API Key – Secret used for authenticating API users, e.g., "5qjEqqufNM5XxXxXxXxXxXxXXXXXXXXXXXX"
* API Endpoint – Specific URL to a function or resource in an API, e.g., "https://assist-azure.openai.azure.com/"
* API Version – Version number or designation of an API, e.g., "2025-04-01-preview"

After creation, any number of LLM models, embedding models, and audio models can be added. These will then be available to users of the AI Chat in a selection menu.

image::../images/Abbildung-24-azure.jpg[Administration - LLMs and Embedding Models - Edit Azure Model, title="Administration - LLMs and Embedding Models - Edit Azure Model", width=500]

For creating AWS Provider Models, the following inputs are required:

* Key Name – Identifying name of an API key, e.g., "aws-key"
* Display Name – User-friendly display name, e.g., "AWS Models" (displayed in the overview in {application})
* IAM Access Key – Username for API access, uniquely assigned identifier, e.g., "AKIAIOSFODNN7EXAMPLE"
* IAM Secret Access Key – Secret part of a key pair, functions like a password, e.g., "wJalrXUtnFEMI/K7MDENG..." (must not be exposed publicly)
* Region – Geographic area/data center where cloud resources run, e.g., "eu-central-1"

After creation, any number of LLM models, embedding models, and audio models can be added. These will then be available to users of the AI Chat in a selection menu.

image::../images/Abbildung-24-local.jpg[Administration - LLMs and Embedding Models - Edit local Model, title="Administration - LLMs and Embedding Models - Edit Azure Model", width=500]

For creating Local Provider Models, the following inputs are required:

* Key Name – Identifying name of an API key, e.g., "development-key"
* Display Name – User-friendly display name, e.g., "Local Models" (displayed in the overview in {application})
* API Key – Secret used for authenticating API users, e.g., "dev-api-key-1234567890abcdef"
* API Endpoint – Specific URL to a function or resource in an API, e.g., "http://localhost:8080/api/v1/users"
* API Version – Version number or designation of an API, e.g., "2026-01-01-latest"

After creation, any number of LLM models, embedding models, and audio models can be added. These will then be available to users of the AI Chat in a selection menu.
