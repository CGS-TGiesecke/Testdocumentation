
=== Authentication

Authentication protects applications and data while fulfilling compliance requirements.
The {application} application supports various authentication methods, which can be configured prior to delivery according to the customer's requirements.

{application} supports:

* OpenID Connect authentication with Microsoft Entra ID
* SAML authentication with Microsoft Entra ID
* Local user/password management

[.highlight]#*Note*: These forms of authentication are fully supported by {application}. Alternative or extended forms are also possible but are not included in the standard scope and are offered separately.#
