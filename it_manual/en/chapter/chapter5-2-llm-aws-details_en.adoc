
===== Step-by-Step: Setting up AWS Bedrock

====== Step 1: Enable AWS Bedrock

1. *Open AWS Console:* (https://console.aws.amazon.com)
2. *Navigate to Bedrock:* Search for "Bedrock"
3. *Select region:* e.g., eu-central-1 (Frankfurt), us-east-1
4. *Enable Model Access:*
   - Navigate to "Model access"
   - Click on "Manage model access"
   - Enable Anthropic Claude models
   - Click on "Save changes"

====== Step 2: Set up IAM permissions

Create an IAM user with the following policy:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "bedrock:InvokeModel",
        "bedrock:InvokeModelWithResponseStream"
      ],
      "Resource": "arn:aws:bedrock:*::foundation-model/*"
    }
  ]
}
```

====== Step 3: Generate Access Keys

1. IAM Console → Users → Your user → Security credentials
2. Create access key
3. *Note for later configuration in the application:*
   - AWS Access Key ID
   - AWS Secret Access Key
   - AWS Region
   - Model ID (e.g., `anthropic.claude-3-5-sonnet-20240620-v1:0`)

*Example:*

```bash
AWS Access Key ID: AKIAIOSFODNN7EXAMPLE
AWS Secret Access Key: wJalrXUtnFEMI/K7MDENG...
AWS Region: eu-central-1
Model ID: anthropic.claude-3-5-sonnet-20240620-v1:0
```
*Dimensioning token rate limits*

Rules of thumb for TPM (Tokens per Minute):

|===
| User Scenario      | Recommended TPM    | Rationale

| 1–5 users (test)   
| 30,000 – 50,000    
| Sufficient for testing

| 10–25 users        
| 80,000 – 120,000   
| Moderate usage

| 25–50 users        
| 150,000 – 250,000  
| Production environment

| 50+ users          
| 300,000+           
| High load
|===

===== Identification with Amazon Resource Name (ARNs)

Amazon Resource Names (ARNs) allow access to a specific Foundation Model (`<model-id>`) in the AWS Bedrock Service (Region: `eu-central-1`).  
The ARN is required to uniquely reference models within applications or pipelines, and to assign permissions for users/AWS services appropriately.

See: https://docs.aws.amazon.com/IAM/latest/UserGuide/reference-arns.html

Example:
```
arn:aws:bedrock:eu-central-1::foundation-model/<model-id>
```

*Structure of the ARN*

- *arn:* AWS resource type identifier ("arn")
- *aws:* Amazon Web Services
- *bedrock:* Service (here: Bedrock)
- *eu-central-1:* AWS region (here: EU [Frankfurt])
- *foundation-model:* Specifies that this is a foundation model
- *<model-id>:* The unique model identifier (e.g., `"cohere.command-text-v14"`)

*Use Cases*

- *Access Control:* Define in IAM policies to assign permissions for specific foundation models.
- *Service Integration:* For using the model via SDK, AWS CLI, or directly in the service.
- *Referencing in Infrastructure Code:* For CDK, Terraform or CloudFormation templates.
