

===== Step-by-Step: Setting up AWS Bedrock

*Step 1: Activate AWS Bedrock*

* Open AWS Console: https://console.aws.amazon.com
* Navigate to Bedrock: enter “Bedrock” in the search bar
* Select region: e.g., eu-central-1 (Frankfurt) or us-east-1
* Enable Model Access:
** Click on *Model access* on the left
** *Manage model access*
** Select required models (e.g., OpenAI OSS, Meta, Anthropic, etc.)
** Click *Save changes*

*Notes:*

* Some 3rd‑party providers (e.g., Anthropic) require 5–15 minutes for activation
* Model Access is mandatory. IAM alone is not sufficient – models must be activated manually.
* Model IDs must match exactly. Incorrect or outdated IDs will cause AccessDeniedException.
* Region must be consistent. Model region, Bedrock API region, and IAM policy must match.

*Step 2: Configure IAM Permissions*

To allow an EC2 instance (or other AWS services) to access Bedrock, either an IAM Role *or* IAM User is required.

* IAM Role (recommended for AWS resources)
* IAM User with Access Keys (recommended for local developer environments)

|===
|Method | Advantages | Disadvantages

|IAM Role
| Automatic, secure temporary credentials via IMDS (Instance Metadata Service), no key management required 
| Only suitable for AWS resources (not for local developers)

|IAM User
| Usable for local tools & developers, even outside AWS 
| Permanent keys must be stored securely & rotated
|===

*Create IAM Role or IAM User*

Create an IAM Role or IAM User and assign the required Bedrock policy. +
Custom IAM policies must be created *before* creating the IAM Role or IAM User, so they can be selected and assigned during creation.

The minimal policy required for Bedrock access is:
```json
{
 "Version": "2012-10-17",
 "Statement": [
   {
     "Effect": "Allow",
     "Action": [
       "bedrock:InvokeModel",
       "bedrock:InvokeModelWithResponseStream"
     ],
     "Resource": "arn:aws:bedrock:*::foundation-model/*"
   }
 ]
}
```
To restrict the policy to a specific region, replace '*' with the region:
```
"Resource": "arn:aws:bedrock:eu-central-1::foundation-model/*"
```

*Info:* Amazon ARNs (Amazon Resource Names) are unique identifiers used to reference AWS resources such as AI models. +
See: https://docs.aws.amazon.com/IAM/latest/UserGuide/reference-arns.html

Example:
```
arn:aws:bedrock:eu-central-1::foundation-model/openai.gpt-oss-20b-1:0
```

*ARN Structure*

* *arn:* Resource type identifier
* *aws:* Amazon Web Services
* *bedrock:* Service (Bedrock)
* *eu-central-1:* AWS region (EU [Frankfurt])
* *foundation-model:* Specifies this is a foundation model
* *<model-id>:*</model-id> The unique model identifier (e.g., openai.gpt-oss-20b-1:0)

*Step 4: Generate Access Keys (not required for IAM Role)*

1. AWS IAM Console → Users → Your user → Security credentials
2. Create access key
3. *Save the following for your application configuration:*
* AWS Access Key ID
* AWS Secret Access Key
* AWS Region 
* Model ID (e.g., openai.gpt-oss-20b-1:0)

Example:
```
AWS Access Key ID: AKIAIOSFODNN7EXAMPLE
AWS Secret Access Key: wJalrXUtnFEMI/K7MDENG...
AWS Region: eu-central-1
Model ID: openai.gpt-oss-20b-1:0
```

*Assigning IAM User / IAM Role to EC2 Instance*

*IAM Role (recommended)*: +
Assign the role to the EC2 instance via → EC2 → Instance → Actions → Security → Modify IAM role.

*IAM User*: +
For an IAM User the AWS credentials at the EC2 instance be stored in the application's configuration.
