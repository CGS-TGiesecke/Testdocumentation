===== Registrierung einer Anwendung in Microsoft Entra ID Admin Center

*Login*

Sign in to the Microsoft Entra ID Admin Center with an account that has administrator rights for your organization.
```
https://entra.microsoft.com
```
*Navigate to "App registrations"*

* In the left menu, select "Identity Management", then "Azure Active Directory".
* Navigate to "App registrations".
* Register a new application
* Click on "New registration" (top of the menu).

*Enter application information*

* Name: Choose a name for the application. *Suggestion:* {application}
* Supported account types: Decide whether the application should support only internal users (your own organization only), external users, or multiple organizations. *Suggestion:* internal users (your own organization only)
* Redirect URI (optional): Enter a redirect URL if your application requires authentication, or specify your app's redirect URI. *Suggestion:* ????????

*Create*

* Click on "Register".
* After registration you will receive:

** Application ID **(Client ID)**
** Directory ID **(Tenant ID)**
** **Client secrets** or certificate as needed under "Certificates & secrets"
```
Note: Please save or copy these values for your application configuration.
```
*Configure permissions (optional)*

* Under "API permissions" you can grant your application access to Microsoft APIs (e.g. Microsoft Graph) (see "Defining SharePoint permissions" below).

*Further settings*

* Optional: Configure branding, token configuration and user assignment as required.
