=== Installation and Configuration

==== Deployment Guide (Step-by-Step)

Deployment of {application} is carried out in the following steps:

1. Prerequisites

- *Docker* and *Docker Compose* are installed.
- The files ``docker-compose.yml`` and ``.env`` are present in the working directory.

2. Login to our DockerHub repository
  
- Use the provided read-access token: ``docker login -u {author} -p [ACCESS_TOKEN]``

3. Configuration Data

- Collect and enter the required data (SharePoint integration, authentication, modules) into `init_config.yaml` (see "Initial Application Configuration")

4. Start the Application

- *Check .env file:* Is the path to ``SHARED_FOLDER`` set correctly?
- Start the application: ``docker compose up -d``

5. Completion
- After installation, restart the web server.
- Test application accessibility in the browser.
- Check status
- Check logs
- Health check

==== Initial Application Configuration

After installation, the following initial configurations must be performed:

*Configure authentication*

- Configure/activate the authentication method chosen by the customer in `init_config.yaml`

*Configure modules*

- LLM API: Configure provider & models in `init_config.yaml`
- RAG API: Set up vector store & embeddings in `init_config.yaml`
- Workflow Engine: Set up automations

==== Update/Upgrade Process

The update and upgrade process, to be performed by the customer, is as follows:

- Before every update, complete backups of the application and database must be created.
- Check the release notes and compatibility lists for the new version.
- Stop the application service or set the application to maintenance mode.
- Replace the application files with the new version according to the update instructions.
- If necessary, execute required database migration steps (see manufacturer's update guide).
- Restart the application and verify correct functionality.

After the update is complete, inform end users and, if necessary, publish a quick guide.
