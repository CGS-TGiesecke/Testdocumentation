=== Introduction

==== Purpose and Scope of the Manual

This manual serves as the central documentation for the installation, configuration, operation, maintenance, and support of the {application} application.
It is intended for IT administrators, system operators, and technical users who are responsible for managing and securely operating the application.

The purpose of this manual is to provide all necessary information in a structured and comprehensible way.

Additionally, the manual helps ensure compliance with internal and external requirements (e.g., data protection, IT security) and serves as a basis for training IT staff.

==== Scope

This manual applies to the {application} application and its use within the customer's IT infrastructure.
It covers all production, test, and development environments in which the application is operated.

The scope includes

- installation and configuration of the application,
- daily operation and maintenance,
- IT security management (especially access and permission management),
- integration with relevant subsystems and interfaces,
- emergency management and disaster recovery.

User-specific configurations, individual customizations as part of special projects, and aspects outside the responsibility of internal IT
(e.g., systems managed by external service providers) are excluded.

==== Target Audience

This manual is primarily intended for IT administrators, system operators, and technical staff of the customer who are responsible for the installation, configuration,
operation, maintenance, and support of the {application} application.

In addition, it serves as a reference for

- IT security officers,
- IT support staff,
- authorized external service providers who maintain or support the application.

The manual is not intended for end users or business departments, but exclusively for technically trained personnel with administrative access rights to the systems.

==== Definitions and Abbreviations

The following terms and abbreviations are central to this manual:

- Application {application}: The AI platform for medium-sized businesses covered in this manual
- Admin: IT administrator with extended access rights to the application and associated systems
- User: General user of the application with restricted rights
- Backup: Data backup for restoration in case of failure or data loss
- Restore: Recovery of backed-up data
- DMS: A Document Management System is a hierarchical document filing system.
- API: Application Programming Interface, an interface for integrating with other systems
- LLM: Large Language Models are language models for text generation
- RAG: Retrieval-Augmented Generation is a software system that processes information from LLMs and generates answers
- MCP: Model Context Protocol is an open-source standard for connecting AI applications with external systems
