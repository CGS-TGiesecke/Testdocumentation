
===== Registering an Application in the Microsoft Entra Admin Center for Mail Integration (Microsoft 365 / Graph)

*Sign in*

Sign in to the Microsoft Entra Admin Center with an account that has administrator rights for your organization.  
`https://entra.microsoft.com`

*Navigate to "App registrations"*

* Select Entra ID on the far left
* Navigate to "App registrations"
* Register a new application
* Click "New registration" (top menu)

*Enter application information*

* Name: Choose a name for the application. The name will be visible later. *Suggestion:* {application}-client-app
* Supported account types: Decide whether the application should support only internal users (single tenant), external users, or multiple organizations. Default is "Single tenant".
* Enter Redirect URIs, must match the "redirect_uri" value in `init_config.yaml` (required for OIDC).

*Create*

* Click "Register"

*For app authentication:*

* Select Certificates & secrets on the left side
* Create a new client secret → copy the value (displayed only once!)
* Or alternatively upload a certificate

*Configuration data:*

** Application ID (under Overview): *Application (client) ID*
** Tenant ID (under Overview): *Directory (tenant) ID*
** Value of the client secret (under Certificates & secrets): *Client Secret Value*, can be time‑limited
** ID of the client secret (under Certificates & secrets): *Client Secret ID*
** ID of the app registration (under Overview): *Object ID*

Note: Record/copy these values for your application configuration in `init_config.yaml`.
