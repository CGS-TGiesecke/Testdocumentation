
=== Operation of the Application by the Customer

==== Starting and Stopping the Application

*Starting the application:*

* Log in to the server with administrative privileges.
* Start the {application} containers.
* Check whether the web server and the database connection are active.
* Test access to the web interface via the browser.

*Stopping the application:*

* Inform users about the planned shutdown.
* Shut down the {application} containers in an orderly manner.
* If necessary, stop scheduled background jobs and back up open database transactions.

==== User Management / Role Management

*User management* is performed according to the selected form of authentication.

|===
|Authentication|Create/edit/delete users

|OpenID Connect authentication with Microsoft Entra ID
|*no*, only via Microsoft Entra Admin Center

|SAML authentication with Microsoft Entra ID
|*no*, only via Microsoft Entra Admin Center. During each login, the user's current profile data is synchronized into {application}.

|local user/password management
|*yes*, only via {application}
|===

*Role management* is performed only via {application}.

==== Application Backup and Recovery

Backup and recovery of the application is performed by the customer. +
SHARED_FOLDER contains the database and configuration files.

==== Monitoring

Monitoring is performed by the customer. +
Log outputs of the application {application} for troubleshooting are configurable (see "Set up environment variables").
