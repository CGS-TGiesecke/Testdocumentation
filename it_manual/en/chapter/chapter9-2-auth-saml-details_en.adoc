===== Registering the application in the Microsoft Entra Admin Center for SAML

*1. Sign in*

* Open the Microsoft Entra Admin Center: https://entra.microsoft.com
* Sign in using an account with administrator privileges for your organization.

*2. Navigate to “App registrations”*

* In the left navigation pane, select Entra ID.
* Open App registrations.
* Click New registration (top menu).

*3. Enter application information*

* Name: Enter a name for the application, e.g.: {application}-SAML-app
* Supported account types: Choose whether the application is intended for internal users only, external users, or multiple organizations. Default setting: Single tenant (Accounts in this organizational directory only).
* Redirect URI: Enter the Redirect URI exactly as defined in the redirect_uri field of your init_config.yaml.

*4. Create the application*

* Click Register.

====== SAML Configuration

*5. Enable SAML as the authentication method*

* Navigate to: Entra ID → Enterprise applications → Your Application → Single sign‑on
* Select SAML as the sign‑on method.
* The SAML-based configuration page will open.

*6. Basic SAML Configuration*

* Identifier (Entity ID): Use the unique identifier shown in the Admin Center.
* Reply/Redirect URL: Must match the value of redirect_uri defined in your init_config.yaml.

*7. Configure Attributes & Claims*

Configure user attributes so that the application {application} receives the necessary user information during each sign‑in.

Ensure the following values are included:

* First name
* Last name
* Email address

This guarantees that the application always receives the most up‑to‑date user data.

*8. SAML Certificates*

* Download the Federation Metadata XML file from the SAML Certificates section.
* Store the file in the SHARED_FOLDER.
* The file path must be identical to the entry idp_metadata_path in your init_config.yaml.

The metadata file contains:

* Entity ID
* SSO URL
* Certificates for signature validation

*9. Collect important configuration data*

[.highlight]#Record the following values for later use for the file init_config.yaml:#

** Application ID (under Overview): *Application (client) ID*
** Tenant ID (under Overview): *Directory (tenant) ID* 
** Client secret(under Certificates & secrets ): *Client Secret*, can be time-limited 
** Redirect‑URI (under Overview): *Redirect‑URI* redirect after successful login to {application}
