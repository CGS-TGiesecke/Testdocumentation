
Um SharePoint-Ressourcen (z. B. Listen, Websites, Dateien) durch eine registrierte Anwendung im Microsoft Entra Admin Center abzudecken, sind weitere zusätzliche Schritte nötig.

===== SharePoint-Berechtigungen verwalten

*Hinzufügen von API-Berechtigungen*

* Navigieren Sie zu Ihrer App-Registrierung:
* Entra Admin Center → Entra ID → Anwendungen → App-Registrierungen → Ihre Anwendung wählen.
* Klicken Sie auf „API-Berechtigungen“.
* Fügen Sie Berechtigungen hinzu:
** Klicken Sie auf "Berechtigung hinzufügen".
** Wählen Sie „Microsoft Graph“
** Sie müssen Anwendungsberechtigungen („Application permissions“) vergeben. Mindestberechtigungen sind "Sites.Read.All" oder "Sites.Selected".

===== Admin-Zustimmung erteilen

Damit Ihre App mit diesen Permissions arbeiten kann, muss ein Mandanten-Administrator die Berechtigungen genehmigen:
In den API-Berechtigungen auf "Admin-Zustimmung für [Ihr Verzeichnis]" klicken.

===== Berechtigung "Sites.Selected" auf SharePoint 

Die Berechtigungen allein auf "Microsoft Graph" reichen nicht aus. Es müssen zusätzlich,
die Berechtigungen explizit auf die einzelnen SharePoint-Sites gewährt werden.
Die Berechtigung "Sites.Selected" erlaubt nur Zugriff auf jene Sites, für die explizit Berechtigungen vergeben wurden.

Wenn die Berechtigungen auf eine einzelne SharePoint-Sites gewährt werden sollen, kann dies direkt über das SharePoint Admin Center oder 
die SharePoint-Benutzeroberfläche erfolgen. 

Wenn viele Sites oder unterschiedliche Berechtigungen verwalten werden müssen, ist das Verwenden des PowerShell-Moduls effizienter.

*Step by Step PowerShell-Modul:* 

*Microsoft Graph PowerShell Modul installieren (falls nicht vorhanden):*
```
Install-Module Microsoft.Graph -Scope CurrentUser
```
Benötigt wird evtl. die Rolle "SharePoint Administrator" oder Globaler Administrator, um die PowerShell-Operationen auszuführen.

*Anmelden:*
```
Connect-MgGraph -Scopes "Sites.Read.All", "Sites.Write.All", "Sites.Manage.All", "AppRoleAssignment.ReadWrite.All"
```

*Die App-ID deiner registrierten Anwendung speichern:*
```
$appId = "IHRE-APP-ID"
```

*SharePoint Site ID herausfinden:*
```
$site = Get-MgSite -Filter "displayName eq 'NAME-IHRER-SITE'"
$siteId = $site.Id
```

*Berechtigung auf der Site gewähren:*

Beispiel zum Gewähren von Read-Berechtigungen:
```
$permission = @{
    Roles = @("read")
    GrantedToIdentities = @(
        @{
            Application = @{
                Id = $appId
                DisplayName = "IHRE-APP-NAME"
            }
        }
    )
}
New-MgSitePermission -SiteId $siteId -BodyParameter $permission
```
