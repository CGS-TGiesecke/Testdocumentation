
===== Applikation registrieren im Microsoft Entra Admin Center für SharePoint

*1. Anmeldung*

* Öffnen Microsoft Entra Admin Center: https://entra.microsoft.com
* Anmeldung mit einem Konto an, das über Administratorrechte verfügt.

*2. Neue App‑Registrierung anlegen*

* Navigieren zu *Entra ID → App‑Registrierungen*.
* Klicken auf *Neue Registrierung*.
* Anwendungsinformationen eingeben:

** *Name*: einen eindeutigen Namen wählen, z. B.: {application}-sharepoint-app
** *Unterstützte Kontotypen*: Wählen wer die Anwendung nutzen darf. Standard: Nur mein Mandant (Single Tenant).
** *Redirect‑URI*: Optional – für SharePoint‑App‑Only Zugriff nicht erforderlich.
* Klicken auf *Registrieren*

*3. App‑Authentifizierung konfigurieren*

* Öffnen *Zertifikate & Geheimnisse*.
* Wählen *Neuer geheimer Clientschlüssel*.
* Kopieren des Secret‑Wert direkt nach dem Erstellen (wird nur einmal angezeigt).
* Alternativ: Hochladen der Zertifikate, falls Zertifikatsbasierte Authentifizierung bevorzugt wird.

*4. Wichtige Konfigurationsdaten erfassen*

[.highlight]#Notieren bzw. kopieren Sie die folgenden Werte für init_config.yaml:#

** Anwendungs-ID (unter Übersicht): *Anwendungs-ID (Client)*
** Mandant-ID (unter Übersicht): *Verzeichnis-ID (Mandant)*
** geheimer Clientschlüssel  (unter Zertifikate & Geheimnisse): *Client-Secrets*, kann zeitlich begrenzt werden

*5. API‑Berechtigungen hinzufügen*

* Öffnen registreierte App → *API‑Berechtigungen*.
* Wählen *Berechtigung hinzufügen → Microsoft Graph → Application permissions*.
** Mindestberechtigungen erforderlich:
** Sites.Read.All (tenantweiter Lesezugriff) oder Sites.Selected (*empfohlen*: Least‑Privilege‑Prinzip)

*6. Admin‑Zustimmung erteilen*

Damit die Anwendung die Berechtigungen nutzen kann:

* Öffnen *API‑Berechtigungen*.
* Klicken auf: *Admin‑Zustimmung für <mandant> erteilen“</mandant>*.
* Bestätigen Freigabe.

*7. SharePoint‑Siteberechtigungen bei Sites.Selected vergeben*

*Wichtig:* „Sites.Selected“ *keinen* tenantweiten Zugriff. +
Die App muss explizit für *jede einzelne Site* freigegeben werden.

Dies kann erfolgen über:

* das *SharePoint Admin Center (UI)*
* *Microsoft Graph PowerShell* (empfohlen bei vielen Sites)
* *PnP PowerShell*

-----------------------------------------------------------------------------------------

*SharePoint Admin Center (UI) Variante*

*1. Öffnen SharePoint Admin Center*

* Gehe zu: https://<tenant>-admin.sharepoint.com
* Anmelden mit mindestens SharePoint Administrator (App-Installationen und Freigaben benötigen Adminrechte.)

*2. Navigieren zu den Apps*

* Links im Menü: Mehr Features 
* Bereich Apps → Öffnen (App-Administration wird im SharePoint Admin Center unter Apps → App-Berechtigungen geführt.)

*3. Öffnen der App‑Berechtigungsübersicht*

* Klicken auf: App-Berechtigungen
* Dort sind alle Apps, die im Tenant installiert wurden – inkl. mehrfacher Einträge, wenn dieselbe App in verschiedenen Sites installiert ist.

*4. Auswahl App, die freigegeben wird*

* Suchen der App in der Liste.
* Klicken auf die Zeile → Details öffnen.

Hier zu sehen:

* auf welchen Sites die App installiert ist
* welche Berechtigungen sie anfragt

*5. Site-spezifische Berechtigungen vergeben*

* Apps mit *Sites.Selected* benötigen expliziten Zugriff pro Site (Standard ist NULL = kein Zugriff).
Dies wird durch Administratoren vergeben.
Die UI‑Freigabe erfolgt sitebasiert:

✅ Schrittfolge:

* Öffnen im SharePoint Admin Center: Aktive Websites
* Auswahl die konkrete Site
* Klick auf: Einstellungen
* Scrolle zu: *App-Berechtigungen / Site-App-Berechtigungen*
(Ableitbar aus: site collection / web / list scopes erscheinen unter „Site App Permissions“.) 
* Dort erscheint die App mit der Anfrage
* Vergib Berechtigungen:
** Lesen
** Bearbeiten
** Vollzugriff

*6. Freigabe bestätigen*

* Speichern Berechtigung
* Die App ist nur für diese eine Site einsatzbereit.

Wenn die App andere Sites benötigt → diesen Vorgang für jede Site einzeln wiederholen.

*Microsoft Graph PowerShell‑Variante* 

*1. Modul installieren*
```
Install-Module Microsoft.Graph -Scope CurrentUser
```
Benötigt wird evtl. die Rolle "SharePoint Administrator" oder "Globaler Administrator".

*2. Anmelden*
```
Connect-MgGraph -Scopes "Sites.Read.All", "Sites.Write.All", "Sites.Manage.All", "AppRoleAssignment.ReadWrite.All"
```

*3. App‑ID festlegen*
```
$appId = "IHRE-APP-ID"
```

*4. Site‑ID ermitteln*
```
$site = Get-MgSite -Filter "displayName eq 'NAME-IHRER-SITE'"
$siteId = $site.Id
```

*5. Berechtigung setzen*
```
$permission = @{
    Roles = @("read")
    GrantedToIdentities = @(
        @{
            Application = @{
                Id = $appId
                DisplayName = "IHRE-APP-NAME"
            }
        }
    )
}
New-MgSitePermission -SiteId $siteId -BodyParameter $permission
```

*PnP PowerShell‑Variante*

*1. Verbinden*
```
Connect-PnPOnline `
    -Url "https://<tenant>.sharepoint.com/sites/<SiteName>" `
    -ClientId "<AppId>" `
    -Thumbprint "<CertThumbprint>" `
    -Tenant "<tenant>.onmicrosoft.com"
```

*2. App bekommt explizit Site‑Berechtigung*
```
$Site = "https://<tenant>.sharepoint.com/sites/<SiteName>"
$AppId = "<AppRegistrationId>"

Grant-PnPAzureADAppSitePermission `
    -AppId $AppId `
    -DisplayName "My Automation App" `
    -Permissions "Write" `
    -Site $Site
```
Mögliche Werte für -Permissions:

* Read
* Write
* FullControl

*3. Berechtigung aktivieren / bestätigen (optional)*

Anzeigen:
```
Get-PnPAzureADAppSitePermission -Site $Site
```
Ändern:
```
Set-PnPAzureADAppSitePermission `
    -AppId $AppId `
    -Permissions "FullControl" `
    -Site $Site
```
