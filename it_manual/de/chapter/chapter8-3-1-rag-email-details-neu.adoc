
===== Applikation registrieren im Microsoft Entra Admin Center für Mail-Integration (Microsoft 365 / Graph)

*1. Anmeldung*

* Öffnen Microsoft Entra Admin Center: https://entra.microsoft.com
* Anmeldung mit einem Konto an, das über Administratorrechte verfügt.

*2. Neue App‑Registrierung anlegen*

* Navigieren zu *Entra ID → App‑Registrierungen*.
* Klicken auf *Neue Registrierung*.
* Anwendungsinformationen eingeben:

** *Name*: einen eindeutigen Namen wählen, z. B.: {application}-email-app
** *Unterstützte Kontotypen*: Wählen wer die Anwendung nutzen darf. Standard: Nur mein Mandant (Single Tenant).
** *Redirect‑URI*: Eintrag Redirect‑URI, die identisch mit redirect_uri in der Datei init_config.yaml ist (für OIDC erforderlich).
* Klicken auf *Registrieren*

*3. App‑Authentifizierung konfigurieren*

* Öffnen *Zertifikate & Geheimnisse*.
* Wählen *Neuer geheimer Clientschlüssel*.
* Kopieren des Secret‑Wert direkt nach dem Erstellen (wird nur einmal angezeigt).
* Alternativ: Hochladen der Zertifikate, falls Zertifikatsbasierte Authentifizierung bevorzugt wird.

*4. Wichtige Konfigurationsdaten notieren*

[.highlight]#Notieren bzw. kopieren Sie die folgenden Werte für init_config.yaml:#

** Anwendungs-ID (unter Übersicht): *Anwendungs-ID (Client)*
** Mandant-ID (unter Übersicht): *Verzeichnis-ID (Mandant)*
** Wert des geheimen Clientschlüssel (unter Zertifikate & Geheimnisse): *Client-Secrets Value*, kann zeitlich begrenzt werden
** ID des geheimen Clientschlüssel (unter Zertifikate & Geheimnisse): *Client Secret ID*
** ID der App-Registrierung (unter Übersicht): *Object ID*

Um E-Mail Ressourcen durch eine registrierte Anwendung im Microsoft Entra Admin Center abzudecken, 
sind weitere zusätzliche Schritte nötig.

====== Microsoft Graph - Berechtigungen verwalten

*5. API‑Berechtigungen hinzufügen*

* Öffnen registreierte App → *API‑Berechtigungen*.
* Wählen *Berechtigung hinzufügen → Microsoft Graph → Application permissions*.
** Mail.Read (ermöglicht der App das Lesen von E‑Mails in Postfächern)

*6. Admin‑Zustimmung erteilen*

Damit die Anwendung die Berechtigungen nutzen kann:

* Öffnen *API‑Berechtigungen*.
* Klicken auf: *Admin‑Zustimmung für <mandant> erteilen“</mandant>*.
* Bestätigen Freigabe.

====== Microsoft Graph Mail Zugriff – RBAC Setup

Um sicherzustellen, dass die Anwendung nur auf bestimmte Postfächer zugreifen kann, muss eine
Application Access Policy mithilfe einer E-Mail‑aktivierten Sicherheitsgruppe eingerichtet sein.

*7. E-Mail‑aktivierte Sicherheitsgruppe erstellen (Exchange Admin Center)*

* Erstellen neue Gruppe:

** Gruppenname: <application>-RBAC-Mailbox-Gruppe
** Gruppen‑E-Mail‑Adresse: <application>-rbac-mailbox@<kundendomain>

*8. Zugangsmailbox zur Gruppe hinzufügen*

* Hinzufügen der Sicherheitsgruppe zum Postfach, auf das die Anwendung zugreifen soll:
** <application>@<kundendomain>

====== PowerShell‑Konfiguration

*8. Verbindung zu Exchange Online herstellen*
```powershell
Connect-ExchangeOnline
```

*10. Application Access Policy erstellen*
```powershell
New-ApplicationAccessPolicy `
  -AppId <APP_CLIENT_ID> `
  -PolicyScopeGroupId "<application>-rbac-mailbox@\<Kundendomain>" `
  -AccessRight RestrictAccess `
  -Description "RBAC Zugriff für <application> Mail Verarbeitung"
```

*11. Policy testen*
```powershell
Test-ApplicationAccessPolicy `
  -AppId \<APP_CLIENT_ID> `
  -Identity <application>@\<Kundendomain>
```

*12. Postfach prüfen*
```powershell
Get-Recipient -Identity <application>@\<Kundendomain> | Format-List Name, RecipientType, PrimarySmtpAddress
```

*Ergebnis:* Die Anwendung kann nur auf Postfächer zugreifen, die sich in der definierten Sicherheitsgruppe befinden.
In unserem Fall: - <application>@\<Kundendomain>
