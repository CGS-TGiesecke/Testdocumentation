
===== Applikation registrieren im Microsoft Entra Admin Center für Mail-Integration (Microsoft 365 / Graph)

*1. Anmeldung*

* Öffnen Sie das Microsoft Entra Admin Center: https://entra.microsoft.com
* Melden Sie sich mit einem Konto an, das über Administratorrechte verfügt.

*2. Navigation zu „App‑Registrierungen“*

* Wählen Sie links im Menü Entra ID.
* Klicken Sie auf App‑Registrierungen.
* Wählen Sie Neue Registrierung.

*3. Anwendungsinformationen eingeben*

* Name: Wählen Sie einen aussagekräftigen Namen, z. B.: {application}-email-app
* Unterstützte Kontotypen: Standard ist: Nur Konten in diesem Organisationsverzeichnis (Single‑Tenant). Wählen Sie andere Optionen nur, wenn Multi‑Tenant‑ oder externe Nutzung erforderlich ist.
* Redirect‑URI: Tragen Sie die Redirect‑URI ein, die identisch mit redirect_uri in der Datei init_config.yaml ist (für OIDC erforderlich).

*4. Anwendung erstellen*

* Klicken Sie auf Registrieren.

*5. App‑Authentifizierung konfigurieren*

* Öffnen Sie im linken Menü Zertifikate & Geheimnisse.
* Klicken Sie auf Neuer geheimer Clientschlüssel.
* Kopieren Sie den Secret‑Wert sofort (wird nur einmal angezeigt!).
* Alternativ: Laden Sie ein Zertifikat hoch, falls Zertifikatsauthentifizierung genutzt werden soll.

*6. Wichtige Konfigurationsdaten notieren*

[.highlight]#Notieren bzw. kopieren Sie die folgenden Werte für init_config.yaml:#

** Anwendungs-ID (unter Übersicht): *Anwendungs-ID (Client)*
** Mandant-ID (unter Übersicht): *Verzeichnis-ID (Mandant)*
** Wert des geheimen Clientschlüssel (unter Zertifikate & Geheimnisse): *Client-Secrets Value*, kann zeitlich begrenzt werden
** ID des geheimen Clientschlüssel (unter Zertifikate & Geheimnisse): *Client Secret ID*
** ID der App-Registrierung (unter Übersicht): *Object ID*

Um E-Mail Ressourcen durch eine registrierte Anwendung im Microsoft Entra Admin Center abzudecken, 
sind weitere zusätzliche Schritte nötig.

====== Microsoft Graph - Berechtigungen verwalten

*7. API‑Berechtigungen hinzufügen*

* Öffnen Sie: Entra Admin Center → Entra ID → Anwendungen → App‑Registrierungen → Ihre Anwendung
* Klicken Sie auf API‑Berechtigungen.
* Wählen Sie Berechtigung hinzufügen.
* Wählen Sie Microsoft Graph.
* Unter Anwendungsberechtigungen (Application permissions) fügen Sie hinzu:
** Mail.Read (ermöglicht der App das Lesen von E‑Mails in Postfächern)

*8. Admin‑Zustimmung erteilen*

* Gehen Sie wieder zu API‑Berechtigungen.
* Klicken Sie auf Admin‑Zustimmung für <ihr verzeichnis=""> erteilen</ihr>.
* Bestätigen Sie die Aktion als Mandanten‑Administrator.

====== Microsoft Graph Mail Zugriff – RBAC Setup

Um sicherzustellen, dass die Anwendung nur auf bestimmte Postfächer zugreifen kann, richten Sie eine
Application Access Policy mithilfe einer E-Mail‑aktivierten Sicherheitsgruppe ein.

*9. E-Mail‑aktivierte Sicherheitsgruppe erstellen (Exchange Admin Center)*

* Erstellen Sie eine neue Gruppe:

** Gruppenname: <application>-RBAC-Mailbox-Gruppe
** Gruppen‑E-Mail‑Adresse: <application>-rbac-mailbox@<kundendomain>

*10. Zugangsmailbox zur Gruppe hinzufügen*

* Fügen Sie der Sicherheitsgruppe das Postfach hinzu, auf das die Anwendung zugreifen soll:
** <application>@<kundendomain>

====== PowerShell‑Konfiguration

*11. Verbindung zu Exchange Online herstellen*
```powershell
Connect-ExchangeOnline
```

*12. Application Access Policy erstellen*
```powershell
New-ApplicationAccessPolicy `
  -AppId <APP_CLIENT_ID> `
  -PolicyScopeGroupId "<application>-rbac-mailbox@\<Kundendomain>" `
  -AccessRight RestrictAccess `
  -Description "RBAC Zugriff für <application> Mail Verarbeitung"
```

*13. Policy testen*
```powershell
Test-ApplicationAccessPolicy `
  -AppId \<APP_CLIENT_ID> `
  -Identity <application>@\<Kundendomain>
```

*14. Postfach prüfen*
```powershell
Get-Recipient -Identity <application>@\<Kundendomain> | Format-List Name, RecipientType, PrimarySmtpAddress
```

*Ergebnis:* Die Anwendung kann nur auf Postfächer zugreifen, die sich in der definierten Sicherheitsgruppe befinden.
In unserem Fall: - <application>@\<Kundendomain>
