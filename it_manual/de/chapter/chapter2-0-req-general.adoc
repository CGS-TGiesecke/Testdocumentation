
=== Allgemeine Vorrausetzungen

==== Infrastruktur (VM oder Server)

Die Lösung wird auf einem *Linux-basierten Server* bereitgestellt (lokale VM oder Cloud-VM). 
Es ist *keine spezielle Cloud-Integration erforderlich* - jeder Linux-Server mit Docker ist geeignet.

*Wichtig:* Eine VM ist *nicht zwingend erforderlich*. Die Lösung kann auf jedem Linux-Server mit Docker betrieben werden.

==== Betriebssystem

* *Linux*, z. B. Ubuntu Server LTS (24.04) oder vergleichbare Distributionen
* Getestet mit Ubuntu; andere Linux-Distributionen auf Anfrage

==== Docker-Umgebung

Die Bereitstellung erfolgt über *Docker*.

*Erforderliche Versionen:*

* *Docker Engine:* Version `>` 2.x

*Installation:* Der Kunde muss Docker auf dem Server installieren und konfigurieren.

Docker speichert Daten (Images, Container, Volumes) standardmäßig unter Linux im Verzeichnis `/var/lib/docker`. 
Volumes für persistente Daten liegen typischerweise in `/var/lib/docker/volumes`.
Für den Betrieb empfehlen wir mindestens 20 GB Speicher für das Docker-Datenverzeichnis zur Verfügung stellen.

==== Gemeinsames Verzeichnis (SHARED_FOLDER)

Ein *beschreibbares Verzeichnis* auf dem Server für den Datenaustausch zwischen den Containern und für persistente Daten (Dokumente, Datenbanken, Vector-Store).

*Aufgabe des Kunden:* 
Bereitstellung eines Verzeichnisses (SHARED_FOLDER) mit ausreichenden Schreib-/Leserechten für die Docker-Container.
Da in diesem Verzeichnis Datenbank und Konfigurationsdateien enthalten sind, ist wachsender Speicherbedarf einzuplanen.
