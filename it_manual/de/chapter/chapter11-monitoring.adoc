
=== Betrieb der Anwendung durch den Kunden

==== Start und Stopp der Anwendung

*Starten der Anwendung:*

* Melden Sie sich am Server mit administrativen Rechten an.
* Starten Sie die {application} Container
* Prüfen Sie, ob der Webserver und die Datenbankverbindung aktiv sind.
* Testen Sie den Zugriff auf die Weboberfläche über den Browser.

*Beenden der Anwendung:*

* Informieren Sie die Benutzer über die geplante Abschaltung.
* Fahren Sie die {application} Container geordnet herunter.
* Beenden Sie ggf. geplante Hintergrundjobs und sichern Sie offene Datenbanktransaktionen.

==== Benutzerverwaltung / Rollenverwaltung 

Die *Benutzerverwaltung* erfolgt nach gewählter Form der Authentifizierung. 

|===
|Authentifizerung|Benutzer erstellen/bearbeiten/löschen

|OpenID Connect Authentifizierung mit Microsoft Entra ID
|*nein*, nur über Microsoft Entra Admin Center

|SAML Authentifizierung mit Microsoft Entra ID
|*nein*, nur über Microsoft Entra Admin Center. Bei jeder Anmeldung werden die aktuellen Profildaten des Benutzers in {application} gespiegelt.

|lokale Benutzer-/Passwortverwaltung
|*ja*, nur über {application}
|===

Die *Rollenverwaltung* erfolgt *nur* innerhalb {application}. 

==== Patch-Management

* Anwendung: {author} stellt Updates bereit und der Kunde spielt ein
* Infrastruktur: Kunde ist verantwortlich (OS, Docker)

==== Backup und Recovery der Applikation

Backup und Recovery der Applikation erfolgt durch den Kunden. +
Im SHARED_FOLDER enthalten sind Datenbank und Konfigurationsdateien.

==== Überwachung/Monitoring

Überwachung/Monitoring erfolgt durch den Kunden. +
Logausgaben der Applikation {application} für Fehlersuche o.ä. sind konfigurierbar (siehe "Umgebungsvariablen einrichten").
