
==== Authentifizierung über lokale Benutzer-/Passwortverwaltung 

Diese Methode wird verwendet, wenn:

* keine externe Identity‑Provider‑Integration gewünscht ist
* ie Verwaltung vollständig innerhalb von {application} erfolgen soll
* ein besonders abgeschottetes Umfeld existiert (z. B. isolierte Netze)

Eigenschaften:

* Benutzer werden in der internen SQLite‑Datenbank verwaltet
* Rollen und Berechtigungen werden in der Administrationsoberfläche gepflegt
* Passwort‑Policies können kundenseitig umgesetzt werden

Diese Methode ist funktional, bietet jedoch weniger Komfort als SSO‑Varianten.
