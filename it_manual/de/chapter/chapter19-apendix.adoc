*Kunde:*

- Alle enthaltenen Komponenten liegen in der Verantwortlichkeit des Kunden.
- Er stellt die VM oder den Server bereit und muss Docker auf dem Server installieren und konfigurieren.
- Er stellt ein Verzeichnis (SHARED_FOLDER) mit ausreichenden Schreib-/Leserechten für die Docker-Container bereit.
- Er stellt den LLM Provider bereit.
- Er konfiguriert die Proxy-Anbindung (TLS/HTTPS)
- Er konfiguriert die notwendigen Firewall Freischaltungen
- Er konfiguriert die Anbindung an die vorhandene Authentifizierung
- Er händelt Backup und Recovery (Datenbank, SHARED_FOLDER , Konfigurationsdateien)

*{application}*

Wir stellen die Anwendung in einem Docker Image bereit. Darin enthalten und über Docker 
installiert sind die einzelnen Container ({application} Backend, LLM API, RAG API, Caddy Proxy, Redis, Celery-Worker, Kundenspezifische MCP Server API)

Wir liefern

- Benutzer-Training 
- Handbücher 
- Konfigurationsanleitungen
- Deployanleitungen

Nach dem Deploy:

- Updates/Patches
- Support
